◆ is_lvalue()

bool ASTType::is_lvalue ( ) const

Unlike other attributes, is_lvalue() and set_is_lvalue() do not look through/modify the canonical type The reason is that, for example, multiple variables all have a type reference to the same struct type, but some of them are lvalues while others aren't, then calling is_lvalue()/set_is_lvalue() should only operate on the type references themselves.
